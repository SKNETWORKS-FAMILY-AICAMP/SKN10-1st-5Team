 ## 🔎 기타
**개발과정에서 발생한 이슈 및 해결방법:** </br>

- **최수헌**</br>
DB를 새로 생성 Table에 데이터를 저장, 파이썬에서 DataFrame으로 받아오는 과정이 쉽지 않았습니다.
  - 새로운 DB를 생성할 때 docker-compose.yml 의 기존에 이미 사용하고 있는 포트를 사용하면 두 서버가 동시에 작동할 수 밖에 없었습니다.</br>
기존에 사용하던 3306:3306 이 아닌, 3307:3306으로 변경하면서 두 서버를 따로 사용할 수 있게 되었습니다.
  - Table에 CSV 데이터를 불러와 저장하는 과정에서, 기존에 있는 테이블에 CSV를 넣으면 기존 테이블의 테이블 설정이 바뀌는 문제가 있었습니다.</br>
이 문제는 정확한 원인은 알지 못했지만, 입력 기존 테이블의 구조와 CSV데이터의 구조를 완벽히 일치시킴으로서 해결되었습니다.
  - 파이썬에서 MySQL 서버에 쿼리한 데이터를 pandas의 DataFrame으로 변환하는 과정에서 모든 데이터가 object 타입으로 바뀌는 문제가 있었습니다.</br>
이 문제도 원인은 알지 못했지만, object 타입으로 바뀐 데이터를 pandas의 astype() 메소드를 통해 다시 변환해 줌으로서 해결했습니다.

- **김현수**</br>
데이터 수집 및 저장에서</br>
첫번째는 js 명령어를 통해 클릭을 해야하는데 제대로 작동되지 않는 문제가 있었고</br>
두번째는 첫번째 이유로 데이터 크롤링이 잘못되었는데 데이터 수만 보고 잘못 수집된 것을 인지하기까지 오래걸렸습니다.</br>
find_element를 사용할 때 문제가 있는 곳이 BY.CLASSNAME을 이용하여 찾는 과정에서의 문제였는데 이를 BY.XPATH로 변경하여 해결했습니다.</br>

**신정우**

프로젝트 시작전에는 streamlit이란 도구로 화면을 구현하는것이 가장 고된 작업이라고 철없이 생각했습니다만,</br>
